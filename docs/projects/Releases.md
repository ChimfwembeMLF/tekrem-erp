# Releases

## What is a Release?
A **Release** is a versioned package of completed work delivered to users or stakeholders. Releases group sprints, epics, and cards for delivery and tracking.

## Use Cases
- Plan and track delivery of new features or fixes
- Group sprints and epics for a version
- Communicate progress and release notes to stakeholders

## User Actions
- Create, edit, and delete releases
- Set release name, version, description, planned/released dates, and status
- Link sprints and epics to a release
- View release progress (story points, percent complete, card status)
- Publish or archive releases
- View all work included in a release

## How Releases Work Together
- Releases group completed work for delivery
- Sprints and epics can be included in releases
- Release progress is tracked by the completion of included work
- Releases help teams plan, communicate, and deliver value incrementally
